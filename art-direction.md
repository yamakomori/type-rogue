# アートディレクション

## アートの役割

画面を飾るためでなく、「落ち着いて次のキーに集中できる場所」をつくる。子ども向けでありながら、キャラクター、背景、UI のどれも幼くしすぎず、余白と素材感を大切にする。

## キーワード

- 静かな夜の海
- 水槽に残る小さな発見
- 深い青と、水面から差すやわらかい光
- ドット絵の魚と、読みやすい現代的なUI
- 余白のある水中図鑑

避けるのは、派手なレア演出、ソーシャルゲーム風の過剰な光沢、情報で埋まった画面、魚との戦闘や逃走を連想させる演出である。

## 配色

基調は低彩度の自然色。地域ごとにアクセントを一色だけ変える。

| 用途 | 色の方向 | 例 |
| --- | --- | --- |
| 背景 | 深い紺、青緑、水面の淡い光 | `#031426`、`#09243A`、`#6EC5CF` |
| 主要文字 | 黒に近い青灰 | `#263238` |
| 正答・進行 | くすんだ青緑 | `#4F8A83` |
| 注意・次キー | やわらかい黄土 | `#D3A64B` |
| 補助 | くすんだ珊瑚、藤色 | 控えめに使用 |

正答とミスを緑・赤だけで区別しない。進捗は形、下線、アイコン、位置でも示す。コントラストは通常テキストで 4.5:1 以上を目安に確認する。

## キャラクターとアイテム

- 主役は小さく丸みのある釣りの相棒。表情は最小限、性別を固定しない。
- 魚は小さなドット絵で、色と輪郭だけでも見分けられる形にする。かわいさよりも静かな生命感を優先する。
- 顔の大きなリアクションより、姿勢・持ち物・周囲の景色で感情を伝える。
- アイテムは帽子、布、植物、道具、光など、現実に近い小さなモチーフから始める。
- 武器は戦闘性より探検道具として扱う。攻撃、勝敗、暴力を連想させる絵は使わない。

## UI の見た目

- 一画面の主役は一つ。タイピング中は問題文と次のキー以外を引っ込める。
- カードは薄い境界線と控えめな影。過度な角丸や光沢を避ける。
- ボタンは色数を絞り、主ボタンを一つだけ強調する。
- 数字は成果を補足する位置に置き、コイン残高・経験値を常時主役にしない。

## 動きと音

- 正答の動きは 120〜180ms 程度の小さな反応。連続打鍵を邪魔しない。
- 地域完了だけは 1〜2秒の静かな演出を許容する。
- 点滅、視差の大きな移動、画面シェイク、強いフラッシュを避ける。
- `prefers-reduced-motion` では動きをフェード中心にし、音も個別にオフにできる。

## アセット制作順

1. タイピング画面のキーボード、指ガイド、相棒、魚影。
2. 水槽の基本背景と、初期3〜5種の魚。
3. 海域ごとの色・珊瑚・海草などの背景差分。
4. 釣果の短い跳ねる演出と、追加コレクション。

初期版ではアニメーションやアイテム数を増やす前に、問題文の可読性と次キーの視認性を実機で確認する。

## 評価

| 観点 | 設計上の扱い |
| --- | --- |
| 学習効果 | 視覚的な競合を抑え、次の一打へ注意を集める |
| 継続率 | 地図と装備の穏やかな変化が愛着になる |
| UX | 色だけに依存せず、動きも控えめで読みやすい |
| 実装コスト | 基本パレット、共通部品、地域アクセントの再利用で進める |
