# 学習分析設計

## 目的と境界

分析は子どもを順位付けするためではなく、次に出す問題を少しだけ良くするために使う。保護者向けにも、他人との比較や長時間プレイを促す数値は表示しない。

## 記録単位

打鍵の生ログを無期限に保存しない。問題終了時に集約した結果と、キー単位の統計だけを保存する。

```ts
type Attempt = {
  id: string;
  problemId: string;
  stageId: string;
  startedAt: string;
  durationMs: number;
  acceptedKeystrokes: number;
  mistakes: number;
  hintCount: number;
  completed: boolean;
  targetKeys: string[];
};

type KeySkill = {
  key: string;
  exposures: number;
  correct: number;
  mistakes: number;
  medianIntervalMs: number | null;
  lastSeenAt: string;
  reviewWeight: number;
};
```

自由入力の本文や個人を特定できる文章は保存しない。保存するなら文字数・完了有無など最小限の匿名集計に限る。

## 指標

| 指標 | 算出 | 用途 | 子どもへの表示 |
| --- | --- | --- |
| キー正確さ | `correct / (correct + mistakes)` | 復習の重み | 原則非表示 |
| キー反応時間 | 正答間隔の中央値 | 難易度調整 | 原則非表示 |
| 習熟度 | 正確さ・反応時間・反復回数の平滑化 | ステージ進行 | 「できるキー」 |
| セッション完了率 | 完了問題 / 開始問題 | 体験の品質確認 | 非表示 |
| 継続日 | プレイした日数のみ | 習慣の把握 | 任意で小さく |

WPM、連続日数、総打鍵数は診断用に保存できても、主報酬・ランキング・通知の軸に使わない。

## キー習熟度

キーごとの正確さは、少ない試行で極端に振れないよう事前値を持たせる。

`accuracy = (correct + 8) / (correct + mistakes + 10)`

反応時間は平均でなく中央値を使い、止まって考えた一回の影響を抑える。`reviewWeight` は以下で上げる。

- 誤入力された。
- 個人の最近の中央値より40%以上遅かった。
- ヒントを出した後に入力した。
- 前回の出題から十分な間隔が空いた。

連続正答で少しずつ下げるが、ゼロにはせず、既習キーも定期的に出す。

## ステージ判定

現在ステージの直近20打鍵（最低8打鍵）を対象にする。

- 正確さが目安以上、かつ新規キーすべてに最低3回の正答があれば「自信あり」。
- 目安を下回っても、規定プレイ数を終えたら次ステージは解放する。
- 「自信あり」でないキーは次のステージでも出題重みを上げる。
- 進捗を失わせたり、過去ステージへ強制的に戻したりしない。

## プレイヤーへのフィードバック

結果画面は一つの達成だけを言葉で返す。

- 「Fキー、すぐ見つけられたね」
- 「さっきより ‘ん’ がスムーズ」
- 「あと1問で新しい道がひらくよ」

「正答率 72%」「平均との差」などの採点感が強い情報は隠す。正確さが低いときは、次の問題を短くすることで支援する。

## 保護者・開発用の集計

保護者画面は任意かつ簡潔にし、週単位で以下だけを示す。

- 遊んだ回数と、集中して終えた短いセッション数。
- 新しく慣れたキー、次に自然に復習するキー。
- 速度比較ではなく「できること」の変化。

開発用の匿名集計では、ステージ別の中断率、ヒント率、入力不能率、問題別のミス率を確認する。特定の問題で中断やミスが偏れば、子どもでなく問題設計を見直す。

## データ保護

- 初期版は端末ローカル保存を基本とする。
- 同期・分析送信を追加する場合は、保護者の明示的な同意、削除、エクスポートを用意する。
- 氏名、年齢、学校、自由入力本文、正確な行動時刻を分析目的で集めない。
- データ保持期間・送信先・用途を子どもにもわかる言葉で説明する。

## 評価

| 観点 | 設計上の扱い |
| --- | --- |
| 学習効果 | 個人のつまずきを、短い復習にそのまま反映する |
| 継続率 | 比較・失格を避け、できた実感だけを返す |
| UX | 数字を読ませず、次の行動がわかる一言に変換する |
| 実装コスト | 問題完了時の集約イベントとキー別集計のみで開始できる |
