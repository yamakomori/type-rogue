# 開発ルール

## コミット

- コミットメッセージは日本語で書く。

## サブエージェントの分担

海域追加など、問題コンテンツとシステム実装の両方を含む機能開発では、原則として以下の役割に分けて進める。ユーザーが別スレッドで依頼した場合も、この分担を適用する。

### 問題担当

- 語彙・例文・学習タグを作成する。
- 対象年齢に合う読みやすさ、自然な日本語、難易度の段階を精査する。
- ローマ字入力、使用キー、表記揺れ、問題の重複を確認する。
- `intro / practice / treasure` など、合意した問題データ形式に従う。

### 実装担当

- ステージ構成、UI、進行、解放条件、セーブ処理を実装する。
- 入力エンジン、問題選択、海域、水槽、魚など、必要なシステムを接続する。
- 問題文の内容は独断で変更せず、問題担当または統合担当へ確認する。

### QA担当

- すべての問題を実際に入力完了できるか確認する。
- 問題ID、入力、語彙、学習タグの重複や不整合を確認する。
- 1プレイ内およびステージ間の難易度を確認する。
- デスクトップと小画面での表示崩れ、進行不能、既存セーブとの互換性を確認する。
- テストとビルドを実行し、問題点を統合担当へ報告する。

### 統合担当（メインエージェント）

- 実装前に仕様、担当範囲、問題データ形式、受け入れ条件を定める。
- サブエージェント同士が同じファイルを同時編集しないよう調整する。
- 問題担当と実装担当の成果を統合する。
- QAの指摘を反映し、最終的なテスト、ビルド、差分レビューを行う。
- 未解決事項や判断したトレードオフをユーザーへ報告する。

## 進め方

1. 統合担当が仕様と問題データ形式を決める。
2. 問題担当と実装担当が、競合しない範囲で並行して作業する。
3. QA担当が統合後の内容を独立して確認する。
4. 統合担当が修正、テスト、ビルド、最終報告を行う。

軽微な文言修正や単一ファイルの小さな変更では、不要な分業は行わない。サブエージェントを使えない環境では、メインエージェントが同じ役割と確認項目を順番に実施する。
