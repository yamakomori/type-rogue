# タイピング判定エンジン設計

## 方針

判定エンジンはゲームの得点装置ではなく、子どもが「次の一打」を理解し、苦手を記録するための基盤である。既存 `/type-app/` の `romajiTable.js` と `romajiMatcher.js` を正本として再利用・移植する。独自のローマ字表を新設しない。

## 必須の挙動

- 1打鍵ごとに正誤と進捗を返す。
- 一般的なローマ字の別綴りを同じ正解として受理する。
- 不正解では入力進捗を戻さず、次の正しいキーを維持する。
- IME を通さず、物理キーボードの `KeyboardEvent.code` を基本に扱う。
- 端末のキーボード配列差・修飾キー・ブラウザショートカットを安全に無視する。
- 成功・ミス・ヒント使用・中断を分析イベントとして返す。

## 受理するローマ字

参照実装に合わせ、次を受理する。

| かな | 受理例 |
| --- | --- |
| し | `shi` / `si` / `ci` |
| ち | `chi` / `ti` |
| つ | `tsu` / `tu` |
| ふ | `fu` / `hu` |
| じ | `ji` / `zi` |
| きゃ | `kya`、単文字入力の連結経路 |
| っ | 次音の子音重ね、`xtu` / `ltu` 等 |
| ん | 文脈に応じた `n`、常に `nn` / `xn` / `n'` |

「正解の綴り」を表示しても、それ以外の許容綴りを不正解にしてはならない。利用者が打った経路に合わせ、表示ヒントもその綴りに追随する。

## 状態モデル

```ts
type TypingState = {
  problemId: string;
  matcher: RomajiMatcher;
  startedAt: number | null;
  acceptedKeystrokes: number;
  mistakes: number;
  mistakeKeys: Record<string, number>;
  keyTimingsMs: number[];
  hintCount: number;
  status: "ready" | "typing" | "cleared" | "abandoned";
};

type KeyResult = {
  accepted: boolean;
  completed: boolean;
  kanaPos: number;
  expectedKey: string;
  event: TypingEvent;
};
```

内部の `matcher` は、複数の入力候補をブランチとして保持する。たとえば `ん` の `n` と後続かなの関係、拗音の2文字・単文字の両経路を、入力途中に破棄しない。確定済みの表示位置には、候補のうち最も前のかな位置を使う。

## イベント処理

1. `keydown` から英数字・必要な記号だけを取り出し、小文字化する。
2. `Ctrl`、`Meta`、`Alt` を伴うもの、移動キー、機能キー、IME 合成中の文字列入力は判定しない。
3. `matcher.handleChar(key)` を呼ぶ。
4. 正解なら進捗・打鍵間隔を記録し、軽い音・視覚反応を出す。
5. 不正解ならミスキーだけを記録し、点滅や大きな赤演出を使わない。
6. 完了なら問題結果を確定し、報酬・学習分析へ1回だけ送る。

入力を `input` イベントの文章から逆算しない。日本語 IME の確定文字列と物理キーは一致しないためである。自動テストやアクセシビリティ入力には、同じ `handleChar` を呼ぶ別アダプターを用意する。

## ヒントとミスの扱い

- 最初の2秒はヒントを出さない。待つことを失敗にしない。
- 3秒間進まない場合、次のキーを淡く強調する。
- さらに5秒後、担当指を表示する。答え全体の自動入力はしない。
- 連続3ミスで問題を中断させず、次のキーだけを静かに案内する。
- ミス回数はコインやアイテムの没収に使わない。

## 完了・中断

| 状態 | 記録 | 画面上の扱い |
| --- | --- | --- |
| cleared | 正答数、ミス、時間、キー別間隔、ヒント | 即座に小報酬へ |
| abandoned | 最終進捗、経過時間、理由 | ペナルティなしで地図へ戻る |
| visibility hidden | 中断開始時刻 | 復帰時に続きから再開 |
| input unavailable | 端末・配列情報 | 設定案内と代替練習 |

## テスト仕様

ロジックは DOM 非依存にし、以下をユニットテストする。

- 基本かな、濁音、拗音、外来音、記号の全テーブル行。
- `し` の `shi/si/ci`、`つ` の `tsu/tu`。
- `きって` の子音重ねと `kitte`、`kixtute` 等。
- `んあ` での `nn` / `n'`、`んか` での裸 `n`。
- 不正キー後も有効な候補と進捗が残ること。
- 同じ問題の完了イベントが二重送信されないこと。
- 日本語配列と US 配列で記号問題を出す前の配列チェック。

コンテンツ追加時は `validateKana` 相当のバリデーターを必ず実行する。

## 評価

| 観点 | 設計上の扱い |
| --- | --- |
| 学習効果 | 正しい入力を即時に強化し、誤入力をキー別に復習へ渡す |
| 継続率 | 別綴りを許し、理不尽な不正解や失敗画面をなくす |
| UX | 現在の入力経路に追随するヒント、最小限のミス反応 |
| 実装コスト | 既存の検証済み純粋ロジックを採用し、UIと分離する |
