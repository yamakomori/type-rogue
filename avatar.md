# アバター・着せ替えシステム設計

## 目的

アバターは勝つための性能ではなく、「自分の冒険を自分らしくする」ための表現である。子どもが初日から選べ、服を得ることが次の短いプレイの楽しみになる構成にする。

## 最小構成

初期版は人型・動物型を選ばせず、一体の中性的な小さな旅人（相棒）を基準にする。体型や性別を選ばせる画面を長く作らず、色と持ち物で個性を出す。

| スロット | 初期所持 | 追加例 |
| --- | --- | --- |
| bodyColor | 6色 | 地域達成カラー |
| head | 3種 | 帽子、花、ヘッドホン |
| outfit | 3種 | マント、雨具、星の服 |
| hand | 2種 | えんぴつ、ランタン、旗 |
| companion | なし | 小鳥、雲、豆ロボット |
| trail | なし | 足あと、葉、光の粒 |

すべて見た目のみで、能力値・得意キー・報酬倍率を変えない。

## 初回体験

1. 相棒の基本色を選ぶ（スキップ可能）。
2. 最初の問題を1問打つ。
3. 帽子または小物をひとつ受け取り、その場で試着する。
4. 地図へ戻る。

名前入力、細かな顔パーツ、プロフィール作成は初期版では行わない。自由入力による個人情報取得も避ける。

## データモデル

```ts
type Item = {
  id: string;
  slot: "bodyColor" | "head" | "outfit" | "hand" | "companion" | "trail";
  name: string;
  price: number;
  theme: "meadow" | "sky" | "cave" | "all";
  layer: number;
  asset: string;
};

type AvatarState = {
  ownedItemIds: string[];
  equipped: Record<Item["slot"], string | null>;
};
```

描画は SVG またはレイヤー化した画像で行い、`layer` の固定順で重ねる。衣装ごとの個別イラストを大量に作る前に、色替え・小物・背景の組合せで十分な変化を出す。

## アクセシビリティと安全

- 色だけでレア度・選択状態を伝えず、輪郭・ラベルを併用する。
- 点滅するトレイルや強い発光は設定で弱められる。
- 肌の色・性別・身体特徴による優劣や、見た目のからかいを連想させる表現を避ける。
- アイテム名は短く読みやすくし、漢字には必要なら読みを添える。

## 評価

| 観点 | 設計上の扱い |
| --- | --- |
| 学習効果 | 装備性能をなくし、学習行動を歪めない |
| 継続率 | 最初から自分で選べ、数プレイで新しい変化が得られる |
| UX | 1画面で試着・装備・戻るが完結する |
| 実装コスト | 6スロット、静的カタログ、重ね描画で始める |
