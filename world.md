# 世界進行・海の設計

## 役割

世界進行はカリキュラムを「次はどの海へ出るか」という冒険に翻訳する。敵、体力、失敗による後退は持たない。タイピングでことばをたどると魚が近づき、1プレイの最後には必ず水槽へ連れて帰れる。

> タイピングで、ことばの海から魚を集める。

## 基本ループ

```text
水槽を眺める
  → 「海をえらぶ」で海域とレッスンを選ぶ
  → 3問または6問を落ち着いて入力する
  → 魚を1匹つる
  → 水槽で新しい魚を見る
```

- 魚は正確さ・速さにかかわらず、1プレイ完了ごとに必ず1匹つれる。
- 同じ海を何度遊んでも、魚は水槽に増える。重複は失敗や「はずれ」と扱わない。
- メダルは釣果の有無を変えない。ていねいさ、速さ、両方達成の金色は、魚の小さな見た目差分として残す。
- 収集の完成や希少な魚を、連続ログイン・期限・課金に結びつけない。

## 海と学習段階

| 海域 | 対応ステージ | 学習上の意味 | 釣れる魚の方向 |
| --- | --- | --- | --- |
| 潮だまり | S00〜S08 | ホームポジションと各キーの位置 | 小さなハゼ、エビ、潮だまりの魚 |
| 浅瀬 | SH01〜SH11 | 母音、各行、濁音、促音・撥音・長音・拗音 | フグ、エイ、ウツボ、クラゲ |
| 珊瑚の森 | CO01〜CO06 | 名詞・動作語・様子語・短いフレーズ | チョウチョウウオ、テンジクダイ、ブダイ、ウミガメ |
| 海の洞窟 | CA01〜CA06 | みじかい文。ようす・場所と時間・接続・理由 | アカマツカサ、イセエビ、ウミシダ、ネコザメ |
| 深海 | 予定 | 長文タイピング | 将来追加 |

海域名とステージ名は、学習内容を完全に隠すためでなく、子どもが「今どこを練習しているか」を穏やかに思い出せる程度に対応させる。

## 1プレイの見せ方

1. 「海をえらぶ」から海域とレッスンを選ぶ。
2. 問題の入力中は、問題文・次キー・指ガイドを主役にする。魚影と糸の進行は脇役であり、入力位置を動かさない。
3. 問題を1つ終えるごとに魚影が少し近づく。
4. そのステージの最後の問題を終えると釣果オーバーレイを出し、魚名と水槽へ帰ることだけを伝える。1プレイの問題数は潮だまりが3問、浅瀬と珊瑚の森が6問、海の洞窟は文の長さに応じて6問から4問へ減らす。
5. 「レッスン一覧へ」「もう1回」は同じ強さで置く。新しい海が開いたときだけ、同じグループに「すすむ」を置く。

速く連打して魚と戦う見せ方、逃した魚、失敗で切れる糸は使わない。

## 水槽

水槽は、集めたものを同時に眺められるホームである。数字の所持量よりも「前よりにぎやか」という継続の実感を返す。

水槽は海域ごとに一つ持つ。現行MVPは潮だまり・浅瀬・珊瑚の森・海の洞窟の水槽を表示し、新しい海域の解放とともに対応する水槽を追加する。

MVPでは以下に限定する。

- 水槽にいる個体と、図鑑で発見済みの種を別々にローカルセーブへ保存する。
- 水槽には最大24匹を同時に表示する。各魚種の新しい個体を先に選び、空いた枠を最近の個体で埋めることで、にぎわいと魚種の多さを両立する。一覧では水槽にいる全個体を管理できる。
- 個体は種、出会った海、見た目差分（通常・ていねい・すばやい・金色）を持つ。
- 解放済みの海だけを対象に、海域・レッスン選択画面と水槽で「出会った魚 / その海の魚」を静かに表示する。未発見の魚は名前を伏せ、次に遊ぶ選択肢を増やすためでなく、水槽での小さな発見として扱う。
- 図鑑には海域ごとの総魚種数と未発見枠を表示する。未発見枠は実際の種の体型を見せず、全種共通の黒い魚影と「？」のドット絵にする。
- 魚は釣れた時と水槽から、任意で海へ逃がせる。逃がした個体は水槽からいなくなるが、図鑑の種の記録は残る。誤操作を避けるため、逃がす前に確認し、同種が複数いる場合は1匹だけ逃がすことを明記する。
- 初めて出会った種には釣果画面で `NEW` を表示する。

水槽の移動、えさ、水草との交換、群れ行動、図鑑の達成率は次段階で検討する。水槽の上限や、一定数の魚を手放す強制は初期版では作らない。

## 評価

| 観点 | 設計上の扱い |
| --- | --- |
| 学習効果 | 成否に関係なく釣れるため、正確な入力を怖がらず反復できる |
| 継続率 | 水槽の変化が毎回視覚的に残り、既習ステージを遊ぶ理由になる |
| UX | 次の行動を「海へ出る」一つに絞り、釣果を短く返す |
| 実装コスト | 魚種・個体差分・海域をデータ化し、まずは静的な水槽表示にする |
