# ゲーム進行・ダンジョン設計

## 役割

世界進行は、カリキュラムを「次にどこへ行くか」という冒険に翻訳するもの。敵を倒す・急かされる・失敗すると戻されるダンジョンにはしない。1回のプレイは、道の先にある小さな発見に近づく体験にする。

## 世界の構成

| 地域 | 対応ステージ | 見た目 | 学習上の意味 |
| --- | --- | --- | --- |
| はじまりの庭 | S00〜S03 | 小さな家と石畳 | ホームポジション |
| かぜの丘 | S04〜S06 | 風車、草、雲 | 上段への移動 |
| ひかりの小道 | S07〜S09 | 夜の草原、灯り | 下段と基本かな |
| ことばの森 | S10〜S12 | 木の実、標識、鳥 | 特殊入力と語彙 |
| 星の図書室 | S13〜S14 | 本、星、窓 | 文と記号 |
| つくる広場 | S15 | 白い机と飾り | 自由入力 |

各地域は3〜5個の短いノードから成る。ノードは問題セットであり、戦闘・体力・敵パラメータを持たない。

## 1プレイの流れ

```text
地図で次の灯りを選ぶ
  → 3〜6問を入力（30〜60秒）
  → 道が少し伸びる／小物を見つける
  → 1つの報酬を受け取る
  → 地図に戻る、または「もう1回」
```

「もう1回」は目立つが、終了ボタンと同じ視認性にする。終了時に確認ダイアログや連続プレイの誘導は出さない。

## ノードの種類

| 種類 | 内容 | 頻度 |
| --- | --- | --- |
| 道しるべ | 新キーを知る導入問題 | 新ステージの最初 |
| 散策 | 通常の短問セット | 基本 |
| おさらい | 苦手キー中心の短問セット | 学習分析に応じて |
| 発見 | 既習中心の宝箱問題 | 1プレイの最後 |
| 景色 | 入力なしで地域の変化を見る | 地域完了時のみ |

`おさらい` は「苦手」という名前で表示しない。「ひかりをたどろう」など、世界観に沿う名称にする。

## 進行ルール

- 最初の未完了ノードをおすすめとして表示するが、解放済みの過去地域は自由に選べる。
- 地域完了は正確さによる星数ではなく、ノード完了数で決める。
- 苦手キーがある場合、次の地域の景色は開きつつ、復習ノードを任意で残す。
- 同じ地域を何度遊んでも、新しい景色・アイテムの小さな変化を得られるが、必須の収集数は増えない。
- 毎日限定の道、時間制の扉、連続ログインの道は作らない。

## 演出

- 正答: 灯りが一つ進む、葉が揺れる程度の短い反応。
- 問題完了: 小物が見つかる、道が一歩描かれる。
- 地域完了: 画面全体を埋めない穏やかな景色変化。
- ミス: 強い揺れ、敵の攻撃、暗転、体力減少は使わない。

## 評価

| 観点 | 設計上の扱い |
| --- | --- |
| 学習効果 | 地域をカリキュラムの順序に対応させ、復習の居場所を作る |
| 継続率 | 毎回短い発見と、地図に残る変化を返す |
| UX | 次にすることは一つ、やめる選択も一つで明確にする |
| 実装コスト | ノード・地域・完了状態をデータ化し、戦闘システムを持たない |
